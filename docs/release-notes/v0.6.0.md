## 🏔️ v0.6.0 — "Above the Clouds"

The icefall is behind us, the ropes are fixed, and the team has pushed through to a vantage point most climbers only dream about. From here you can see the entire route — every ridge, every col, every camp below and above. That's exactly what this release delivers: unprecedented visibility into your running apps with the DevFlow Inspector, two brand-new platform targets, a rebuilt dashboard with live device tracking, and a whole new publishing workflow. The pack is heavier, but the view? Absolutely worth it.

### 🆕 New Features

- **DevFlow App Inspector** — The biggest single feature since launch. A full remote inspector for running .NET MAUI apps: browse the visual tree with VS Code–style indent guides, click anywhere on a live screenshot to identify elements via hit testing, edit properties in real-time with type-aware editors (colors, gradients, thicknesses, enums), inspect network requests in a sortable DataGrid, debug Blazor WebViews via CDP with DOM snapshots and JS evaluation, and stream application logs over WebSocket — all without touching the device. (#82, #84)
- **Native macOS (AppKit) App** — A fully native macOS head project with NSToolbar, NSSplitView sidebar, window and sidebar persistence, and platform-native search. No more Mac Catalyst pretending to be a Mac app — this one has real crampons.
- **Linux GTK App** — MAUI Sherpa now runs on Linux via a GTK4 head project with a native HeaderBar toolbar, dark mode via GSettings monitoring, and full feature parity with the Blazor UI. Ships as AppImage, Deb, and Flatpak.
- **Dashboard Overhaul** — The Dashboard got a complete makeover: "At a Glance" cards, embedded release notes, and live device monitoring for connected Android and Apple devices with one-click inspector launch from device cards.
- **Publish Profiles** — A new service for managing app publishing configurations: multi-publisher dialogs, key mappings, an Apple config wizard, review dialogs, and per-profile cert sync status badges.
- **Copilot Native Modal** — The Copilot chat has been lifted out of its Blazor overlay into a dedicated BlazorWebView on its own native modal — snappier, more reliable, and no longer fighting for z-index territory.
- **Vaultwarden / Bitwarden Cloud Secrets** — A new cloud secrets provider backed by Vaultwarden's REST API, so you can sync keystores and signing secrets through your own self-hosted vault. (#59)
- **Push Projects** — Manage push notification project configurations with backup/restore support and direct integration into the Push Testing and Firebase Push pages.
- **macOS In-App Auto-Update** — The macOS app can now check for, download, and install updates on its own — no more manual DMG hunts.
- **GitHub Pages Website** — A full marketing site with a landing page, getting-started guide, DevFlow documentation, and a dark-mode screenshot gallery. Mountain divider SVG included.

### ✨ Improvements

- **Platform-Colored Agent Cards** — DevFlow agent cards now show platform-specific squircle icons with brand color overlays (Apple grey, Android green, Windows blue, Linux orange) and device-type indicators.
- **Demo Mode Blur** — Sensitive Apple data (identity names, key IDs, certificate names, profile names, device names, bundle IDs) is now blurred across all pages when Demo Mode is enabled.
- **Inline Modals → Native Hybrid Modals** — Publisher dialog, Apple config wizard, and secret picker migrated from Blazor inline modals to native platform sheets for a more polished feel.
- **Reusable DataGrid Component** — A new `DataGrid.razor` component with resizable columns, multi-column sorting, text filtering, and custom cell templates — powering the network inspector and ready for future pages.
- **Toolbar Search Everywhere** — Toolbar-driven search now works on Emulators, Keystores, and all list pages with consistent empty-state messaging.
- **Settings Backup Enhancements** — Google identities are now included in backup exports, with full selection metadata and cloud provider configurations.
- **Sidebar & Window Persistence** — Sidebar width and window size/position persist across app launches on macOS.
- **Agent Version Enforcement** — The inspector displays the connected agent's version and blocks interaction if it's below the minimum supported version.

### 🐛 Bug Fixes

- **xcdevice Process Leak** — `xcdevice list` and `xcrun simctl list` processes were accumulating over time. All spawned processes now get cancellation-triggered kills and a 10-second hard timeout.
- **Inspector DPI Miscalculation** — Opening a modal page caused the DPI to spike because `FindFirstWindowBounds` returned a tiny synthetic element. Replaced with `FindLargestWindowBounds`.
- **dotnet Not Found on Doctor Fix** — Fixed the Doctor page failing to fix workloads because `dotnet` wasn't on the resolved PATH.
- **Copilot CLI Not Found in Release Builds** — Fixed the CLI binary not being located on Mac Catalyst release builds. (#81)
- **Azure Key Vault Secret Filtering** — Fixed secret list not being retrieved due to overly aggressive filtering. (#85)
- **macOS App Crash on Launch** — Added the missing `com.apple.security.cs.allow-jit` entitlement that caused `xamarin_vm_initialize` to abort.
- **Android Shell Channel Crash** — Fixed the Android device shell closing prematurely when the stderr reader completed before stdout.

### 🔧 Infrastructure

- **MauiDevFlow 0.18.0** — Updated all agent packages and the CLI tool to 0.18.0, bringing stable element IDs, `windowBounds` data, improved synthetic element support, and cleaner log source prefixes.
- **CI/CD for All Platforms** — GitHub Actions now builds and publishes native macOS (AppKit), Linux (x64 + arm64), and Mac Catalyst apps with signing, notarization, and artifact upload.
- **Linux Packaging Pipeline** — AppImage, Deb, and Flatpak packages built natively for both x64 and arm64 architectures.
- **Platform.Maui.MacOS 0.3.0** — Updated macOS platform packages with stacking sheet support, modal resize fixes, and toolbar visibility improvements.
- **GitHub Pages Workflow** — New `pages.yml` workflow deploys the website from the `website/` directory on pushes to `main`.
